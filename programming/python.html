CAT

I have been sceptical about using a mouse after a mouse related injury back in 1988 and in 1995 I dreamed up an alternative, CAT (Cross Aiming Tool). I made some mock-ups, but I wasn't a good enough programmer to make an actual CAT until recently. I'm not using it myself, partly because I'm using dual screens these days and partly because I have grown used to using a mouse.

The code for CAT can be found on Github

I learned a lot about windows management during the project. Among other things that transparent windows are tough for most languages. Even Pythons Tcl can only manage semi-transparency.


Games

Toke (my youngest) and I made a couple of games while leaning to program in Python.

Caterpillar is a &34frac; finished game involving a caterpillar moving between planets, landing on planets and eating coloured blocks. Basically an attempt on a 3D Snake clone.

Another game was Two Player Tetris where two players cooperate to complete lines on a Tetris board.


A third game was BlockJoin, an attempt to merge a 3D Tetris with Shokoban.


I have always disliked the way calendars are organised: Time is linear to me, not zig-zaggy like a normal month view. Timer Machine is an atttempt to make a 3D layout for calendars. Maybe I'll play more with the concept some day. I still like the look. Trouble is that the 3D package I used isn't too hot on big chunks of text. Maybe HTML5 can be bullied into doing something along these lines.
